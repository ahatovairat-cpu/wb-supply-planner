# WB Supply Planner

Локальное веб-приложение для планирования закупок на Wildberries.

## Что умеет

- Загружает продажи, остатки, заказы и габариты товаров через WB API
- Показывает план закупок: сколько и когда заказывать по каждому артикулу
- Анализирует локализацию: откуда идут заказы на каждый артикул и сколько дней продаж осталось на каждом складе
- Рекомендует оптимальное распределение товара по складам WB с учётом:
  - % локализации (откуда идут заказы)
  - стоимости хранения на складе WB
  - коэффициента логистики WB
  - процента возвратов
- Учитывает логистическое плечо (сколько дней идёт поставка) при расчёте плана

## Установка и запуск

### 1. Установите зависимости
```
pip3 install -r requirements.txt
```

### 2. Запустите приложение
```
python3 app.py
```

### 3. Откройте браузер
```
http://localhost:5000
```

## Как пользоваться

1. **Подключение** — введите API-токен WB (раздел «Статистика»), выберите период и нажмите «Подключить»
2. **Склады** — проверьте и скорректируйте параметры складов (стоимость хранения, коэф. логистики WB, % возвратов)
3. **Товары** — укажите себестоимость каждого товара для точного расчёта суммы закупки
4. **План закупок** — настройте параметры (логистическое плечо, страховой запас, горизонт) и нажмите «Пересчитать»
5. **Локализация** — смотрите по каждому артикулу и складу: остаток, % заказов, сколько дней продаж осталось

## Формула расчёта оптимального склада

Рейтинг склада для каждого товара:
- 40% — % локализации (доля заказов с этого склада)
- 30% — стоимость хранения (меньше = лучше)
- 20% — коэффициент логистики WB (меньше = лучше)
- 10% — % возвратов (меньше = лучше)

Распределение товара по складам пропорционально рейтингу × % локализации.

## Формула плана закупок

```
Нужно заказать = (горизонт + логистическое плечо + страховой запас) × продаж/день − текущий остаток
```

Если значение < 0 — заказ не нужен.
Дата «заказать до» = сегодня + (дней остатка − плечо − страховой запас)
